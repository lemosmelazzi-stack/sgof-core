Mapa base del backend SGOF
1. Módulo autenticación

Se encarga de entrar al sistema y validar usuarios.

Endpoints base

POST /auth/login

POST /auth/logout

GET /auth/me

Qué hace

valida usuario y contraseña

devuelve sesión o token

identifica rol y empresa

2. Módulo empresas

Administra la empresa y su configuración.

Endpoints base

GET /empresas/:id

POST /empresas

PATCH /empresas/:id

GET /empresas/:id/configuracion

PATCH /empresas/:id/configuracion

Qué hace

crea empresa

consulta datos

actualiza configuración general

3. Módulo usuarios

PATCH /usuarios/:id/desactivar

activación/desactivación

Flujo operativo mínimo del backend
Flujo 1 — Pedido manual

POST /pedidos

buscar chofer/taxi disponible

POST /asignaciones

PATCH /pedidos/:id/estado

POST /viajes

Flujo 2 — Inicio de viaje

POST /viajes/:id/iniciar

PATCH /choferes/:id/estado

PATCH /taxis/:id/estado

Flujo 3 — Finalizar viaje

POST /viajes/:id/finalizar

PATCH /choferes/:id/estado

PATCH /taxis/:id/estado

POST /pagos

Lo mínimo para que “funcione”

Si querés un MVP backend de verdad, lo mínimo mínimo es:

POST /auth/login

POST /pedidos

POST /asignaciones

POST /viajes

POST /viajes/:id/finalizar

POST /pagos

GET /viajes

GET /pedidos

Con eso ya tenés operación básica.
